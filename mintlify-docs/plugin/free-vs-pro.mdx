## How tier gating works

The `SessionStart` hook reads `~/.claude/commander/license.json` and sets `CCC_TIER` for the session. Pro-only features check this flag and degrade gracefully on Free:

- Pro hooks no-op instantly with `{ continue: true, suppressOutput: true }`
- Pro skills show an upgrade nudge instead of executing the gated flow
- Agent delegation falls back to in-skill execution

No network calls. License verification is **100% offline** via HMAC signature.

## Upgrading

<Steps>
  <Step title="Subscribe">
    Visit [cc-commander.com/pricing](https://cc-commander.com/pricing) and pick Pro or Team.
  </Step>
  <Step title="Check your email">
    Your license key arrives at the email you used for checkout. Format: `ccc_live_<token>.<signature>`.
  </Step>
  <Step title="Activate">
    ```bash
    cat > ~/.claude/commander/license.json <<'EOF'
    {
      "key": "ccc_live_xxx.yyy",
      "tier": "pro",
      "email": "you@example.com",
      "expires": "2027-04-16T00:00:00Z"
    }
    EOF
    ```
  </Step>
  <Step title="Verify">
    Start a new Claude Code session. The `SessionStart` hook prints:
    ```
    CCC 3.0 | Sessions: X | Knowledge: Y entries | Tier: pro
    ```
  </Step>
</Steps>
