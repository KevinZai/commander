CC Commander v4.0.0-beta.10 ships as a native Claude Code Desktop plugin. No npm install required.

## Install

### Via Desktop GUI (recommended)

The primary install path — works in Claude Code Desktop and Cowork Desktop without opening a terminal.

<Steps>
  <Step title="Open Plugin Marketplace">
    In Claude Code Desktop or Cowork Desktop: **Settings → Plugin Marketplace**
  </Step>
  <Step title="Add from GitHub">
    Click **Add from GitHub** → enter `KevinZai/commander` → click **Add**
  </Step>
  <Step title="Install the plugin">
    Find **commander** in the marketplace list → click **Install**. Takes a few seconds.
  </Step>
  <Step title="Open the hub">
    ```
    /ccc
    ```
    The hub opens with a chip picker. If you see the intent tiles, you're live.
  </Step>
</Steps>

### Via CLI (Claude Code terminal only)

`/plugin` commands are only available in the Claude Code terminal — not in the Desktop chat interface.

<Steps>
  <Step title="Add the marketplace">
    ```
    /plugin marketplace add KevinZai/commander
    ```
  </Step>
  <Step title="Install the plugin">
    ```
    /plugin install commander
    ```
  </Step>
  <Step title="Verify">
    ```
    /ccc
    ```
  </Step>
</Steps>

## Verify

After install:

```
/ccc
```

Expected: the hub opens with 6 intent chips (Build, Review, Ship, Design, Learn, More). All 33 skills active. Zero config required.


## Uninstall

```
/plugin uninstall commander
```

This removes the plugin. Your Claude Code settings and CLAUDE.md files are not affected.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Plugin not found in marketplace">
    Make sure you ran `/plugin marketplace add KevinZai/commander` first. The marketplace must be registered before `/plugin install commander` will work.
  </Accordion>
  <Accordion title="/ccc command not recognized">
    Try restarting your Claude Code session after install. The plugin registers commands at session start.
  </Accordion>
  <Accordion title="Skills not loading">
    Run `/ccc --status` to check the plugin state. If skills show 0, run `/plugin reinstall commander`.
  </Accordion>
</AccordionGroup>
