CC Commander v4.0.0-beta.11 ships **51 plugin skills** across six groups. All skills are **free forever** — no caps, no call limits, no tier gating. All skills appear as plain `/ccc-*` in autocomplete — no namespace prefix required. Run `/ccc-browse` in Desktop for the live filterable catalog.

## Group 1 — 12 /ccc-* Specialist Workflows

Click-first workflows. Each opens an interactive picker (AskUserQuestion native chips) — no flags or arguments needed.

| Skill | Trigger | What it does |
|-------|---------|--------------|
| `/ccc` | `/ccc` or "open commander" | Main hub — detects project context, routes to the right specialist workflow via nested picker |
| `/ccc-start` | `/ccc-start` or "get started with ccc" | First-run onboarding — creates plan file, explains the plugin, surfaces your first win |
| `/ccc-browse` | `/ccc-browse` or "browse skills" | Searchable catalog of all 502+ skills and 17 agents — filter by domain, keyword, or tier |
| `/ccc-plan` | `/ccc-plan` or "plan a feature" | Spec-first planning — 5-question interview → `tasks/spec-YYYYMMDD.md` → ready to build |
| `/ccc-build` | `/ccc-build` or "build something" | Guided build wizard — web app, API, CLI, or mobile. Dispatches the builder agent with spec |
| `/ccc-review` | `/ccc-review` or "review my code" | Branch audit — diff review, security pass, performance check, or full x-ray. Uses reviewer agent |
| `/ccc-ship` | `/ccc-ship` or "ship this" | Pre-flight checks + release + deploy. 22-point gate: TypeScript, tests, secrets, env vars |
| `/ccc-design` | `/ccc-design` or "design this" | UI/UX workflow — routes to ccc-design domain (39 skills: animations, systems, accessibility) |
| `/ccc-learn` | `/ccc-learn` or "teach me" | Skill discovery across all 11 CCC domains — explains what's available and when to use each |
| `/ccc-xray` | `/ccc-xray` or "audit my project" | Project health scorecard — surfaces issues as fix chips you can click to resolve |
| `/ccc-linear` | `/ccc-linear` or "show my board" | Linear board integration — view issues, pick one to build, create new issues, sync status |
| `/ccc-fleet` | `/ccc-fleet` or "run in parallel" | Multi-agent orchestration — launch, route, monitor, and kill parallel agent workers |
| `/ccc-connect` | `/ccc-connect` or "connect my tools" | Opt-in MCP connector — Notion, Zapier, Supabase, Slack, Google Drive, Figma, and more |

## Group 2 — 14 ccc-* Domain Routers

Load ONE domain skill to get an entire specialty. Each router matches your intent and dispatches to the right sub-skill.

| Skill | Skills Inside | What it covers |
|-------|--------------|----------------|
| `ccc-design` | 39 | UI/UX, animations, visual effects, design systems, accessibility, landing pages |
| `ccc-marketing` | 45 | CRO, email campaigns, ads, social content, growth, sales copy |
| `ccc-saas` | 21 | Auth, billing, multi-tenant DB, API patterns, frontend stack, SaaS metrics |
| `ccc-devops` | 21 | CI/CD, Docker, AWS, Terraform, monitoring, zero-downtime deploys |
| `ccc-seo` | 20 | Technical SEO, Core Web Vitals, schema markup, AI search, programmatic SEO |
| `ccc-testing` | 15 | TDD, E2E (Playwright), visual testing, load testing, coverage audits |
| `ccc-data` | 8 | SQL, ETL pipelines, data warehousing, analytics, visualization, ML ops |
| `ccc-security` | 8 | OWASP audits, pen testing, secrets management, supply chain, threat modeling |
| `ccc-research` | 8 | Deep research, competitive analysis, literature review, data synthesis |
| `ccc-mobile` | 8 | React Native, Expo, iOS, Android, Flutter, app store optimization |
| `ccc-makeover` | 3 | `/xray` project audit + `/makeover` design swarm + report generation |

> **Note:** The domain routers for `ccc-design`, `ccc-fleet`, and `ccc-linear` are also exposed as `/ccc-design`, `/ccc-fleet`, and `/ccc-linear` specialist workflows (Group 1). They appear in both groups because they serve double duty — as click-first entry points and as loadable domain packs.

## Group 3 — 2 Diagnostic / Meta Skills

| Skill | Trigger | What it does |
|-------|---------|--------------|
| `/ccc-suggest` | Auto-triggered at session start (configurable) | Ambient intelligence — suggests the best next skill based on project context, recent activity, and active tasks. 4 involvement levels: silent, suggest, prompt, auto |
| `/ccc-cheatsheet` | `/ccc-cheatsheet` or "show cheatsheet" | Inline cheatsheet — canonical counts, all /ccc-* commands, domain list, hook events. Quick reference without leaving the session |

## Group 4 — 2 Vendor-Sourced Skills (beta.9)

Adapted from the superpowers vendor with attribution. Both are free forever.

| Skill | Trigger | What it does |
|-------|---------|--------------|
| `/ccc-agent-writing` | `/ccc-agent-writing` or "write a spec", "improve this doc" | Teaches agents to write specs, PRDs, PR descriptions, and docs with clarity, persuasion, and testability. Delegates long-form content to `content-strategist` persona. |
| `/ccc-systematic-debugging` | `/ccc-systematic-debugging` or "debug this", "root cause", "investigate" | Enforces the Iron Law: no fix without confirmed root cause. Reproduce → hypothesize → verify → fix. Delegates deep investigations to `debugger` persona. |

## Group 5 — Session + Lifecycle Skills (beta.10 + beta.11)

Added in v4.0.0-beta.10 and v4.0.0-beta.11.

| Skill | Trigger | What it does |
|-------|---------|--------------|
| `/ccc-e2e` | `/ccc-e2e` or "e2e test", "add playwright tests" | E2E test scaffolding and Playwright automation — new test suite, existing project, or visual regression |
| `/save-session` | `/save-session` or "save session", "save my progress" | Save current session state to `~/.claude/commander/sessions/` — context, todos, and plan preserved |
| `/resume-session` | `/resume-session` or "resume session", "pick up where I left off" | Resume a previously saved session — lists recent saves, pick one to restore context |
| `/ccc-changelog` | `/ccc-changelog` or "what changed", "what is new" | Renders the latest CHANGELOG.md section. Fires automatically on version-transition via SessionStart nudge (beta.11) |
| `/ccc-doctor` | `/ccc-doctor` or "diagnose plugin" | Plugin diagnostic — audits hooks, MCPs, permissions, frontmatter, test status. Reports green/yellow/red per component (beta.11) |

## Group 6 — Memory / Task / Recall (MIT-ported from community)

Added in beta.10. MIT-licensed ports with attribution.

| Skill | Trigger | What it does |
|-------|---------|--------------|
| `/ccc-memory` | `/ccc-memory` or "remember this" | Two-tier persistent memory: CLAUDE.md (always-loaded) + `memory/` dir (dated notes, decisions, corrections) |
| `/ccc-tasks` | `/ccc-tasks` or "what's on my list" | Lightweight markdown task tracking via `tasks/todo.md`. No database, no service — just checkboxes + priorities |
| `/ccc-recall` | `/ccc-recall` or "what did we do last time" | Search across past Claude Code sessions — find when you solved a similar problem or made a decision |
