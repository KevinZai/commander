## The `~~category` placeholder pattern

CCC skills reference tools **generically** using `~~category` placeholders. Each skill is tool-agnostic — "post to ~~chat" resolves to Slack if you have Slack connected, Teams if you have Teams, or falls back to manual instructions if nothing is connected.

See [CONNECTORS.md](https://github.com/KevinZai/commander/blob/main/commander/cowork-plugin/CONNECTORS.md) for the full category map.

## Tavily setup

Tavily is the only pre-configured server that needs an API key:

<Steps>
  <Step title="Get a key">
    Sign up free at [tavily.com](https://tavily.com). Free tier: 1,000 searches/month.
  </Step>
  <Step title="Set the env var">
    ```bash
    export TAVILY_API_KEY=tvly-xxx
    ```
    Add to `~/.zshrc` or your shell config so it persists.
  </Step>
  <Step title="Verify">
    Start a new Claude Code session and try `/ccc:research tell me about the latest Claude Code release`. Tavily will fetch live results.
  </Step>
</Steps>

## Standalone + supercharged

Every CCC skill works **standalone** (no MCPs required) and gets **supercharged** when connectors are present. Example: `/ccc:standup` works with just git history, but becomes dramatically better with Linear + Slack connected.

## Adding your own MCP servers

You can add more MCP servers at the user level (`~/.claude/settings.json`) without modifying the plugin. CCC automatically detects and uses any connected server matching its `~~category` placeholders.

Popular additions:
- **Notion** — `~~knowledge base`
- **Typefully** — `~~social media`
- **Raindrop** — `~~bookmarks`
- **PagerDuty** — `~~incident management`
- **Datadog** — `~~monitoring`
