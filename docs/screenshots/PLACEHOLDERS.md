# Screenshot Briefs — CC Commander v4.0

Ideal resolution: **2560x1600** (Retina MBP) or **1920x1200** (external display). PNG, no compression. Crop to the Desktop app window — no OS chrome unless the menu bar is relevant.

---

## 01-install-marketplace.png

**What to capture:** The Add from GitHub dialog in Plugin Marketplace. The field should show `KevinZai/commander` already typed. The blue "Add" button should be visible and not yet clicked.

**Why it matters:** First install is the highest-friction moment. Showing the exact field label and URL format removes the guesswork.

**Frame notes:** Full Desktop window. Settings panel open on the left, Add from GitHub modal centered. If the modal overlays other content, that's fine — it shows this is native, not a CLI thing.

---

## 02-plugin-installed.png

**What to capture:** Settings → Plugins tab. The `commander` entry showing version `4.0.0-beta.9` (or current), with the enable toggle ON (purple/blue = enabled).

**Why it matters:** Confirms what "installed" looks like. Reduces support questions about whether it worked.

**Frame notes:** Crop to just the Plugins list area. The `commander` row should be the top visible entry. Show at least one other plugin row below to prove this is a real list.

## 03-ccc-hub-menu.png

**What to capture:** The main `/ccc` hub mid-display — AskUserQuestion chip picker showing 6 intent chips (Plan, Build, Review, Ship, Learn, Fleet or equivalent). The input bar should show `/ccc` with the cursor.

**Why it matters:** This is the hero demo frame. It shows the click-first UX — no typing required after `/ccc`.

**Frame notes:** Full Desktop window so the chip picker's visual weight reads clearly. Dark mode preferred (purple chips pop).

## 04-ccc-browse-catalog.png

**What to capture:** `/ccc-browse` skill catalog — the filterable grid showing at least 8 skill chips. Category filter chips visible at the top (e.g., "Planning", "Build", "Review").

**Why it matters:** Shows breadth without needing to enumerate 30 skills in prose.

**Frame notes:** Scroll far enough to show the category filter row + at least 2 rows of skill chips below. The `/ccc-browse` prompt label should be visible.

## 05-ccc-plan-in-action.png

**What to capture:** `/ccc-plan` mid-flow — ideally the moment a plan file is visible in Desktop's right-side Plan pane. The left chat panel should show the `/ccc-plan` conversation with a few AskUserQuestion steps answered.

**Why it matters:** Demonstrates the plan-first workflow that separates CCC from raw skills.

**Frame notes:** Show both the chat panel (left) and the Plan pane (right) in the same frame. The plan content doesn't need to be real — a dummy project plan is fine.

## 06-ccc-fleet-parallel.png

**What to capture:** `/ccc-fleet` after dispatching 3 sub-agents — the background task status UI (sidebar or status chips) showing 3 agents running in parallel.

**Why it matters:** Parallel orchestration is the power-user hook. One frame sells the concept better than 200 words.

**Frame notes:** The agent labels should be readable (e.g., "Agent 1: security audit", "Agent 2: test coverage"). If Desktop shows a spinner or status badge per agent, make sure at least 2 are visible.

## 07-ccc-start-onboarding.png

**What to capture:** `/ccc-start` welcome screen — the ASCII CCC logo rendered in the chat panel, plus the first onboarding prompt asking the user's project type or skill level.

**Why it matters:** First-run experience matters. This screenshot reassures new installs that something visual and welcoming happens immediately.

**Frame notes:** Full chat panel, no need to show settings or sidebar. Dark mode + the purple/white ASCII logo contrast is the money shot.
