CC Commander registers **8 lifecycle hook events** with **16 total handlers** that fire automatically around every Claude Code session. All hooks are free — no tier gating, no feature flags.

All hooks use `${CLAUDE_PLUGIN_ROOT}` for portable path resolution and fail gracefully — if a hook errors, the session continues uninterrupted.

<Note>
  **v4.0.0-beta.11:** `SessionStart` now uses a single-process orchestrator (`session-start-orchestrator.js`) that runs all 4 session-start handlers in one Node.js process, saving ~300ms of cold-start overhead per session. The 16-handler count is unchanged — the orchestrator is a performance optimization, not a feature reduction.
</Note>

## Hook Registry

| Event | Handlers | Scripts | Purpose |
|-------|----------|---------|---------|
| `SessionStart` | 4 (via orchestrator) | `session-start-orchestrator.js` → `session-start.js`, `stale-claude-md-nudge.js`, `post-compact-recovery.js`, `suggest-ticker.js` | Seed state, warn on stale CLAUDE.md, restore post-compact context, surface ambient next-step suggestion |
| `UserPromptSubmit` | 4 | `suggest-ticker.js`, `intent-classifier.js`, `context-warning.js`, `user-prompt-submit.js` | Ambient suggestion tick; match prompt against skill catalog; warn at 80%/95% context; generic prompt dispatcher |
| `PreToolUse` | 3 | `cost-tracker.js`, `cost-ceiling-enforcer.js`, `secret-leak-guard.js` | Accumulate session cost; block calls over ceiling; scan args for leaked secrets |
| `PostToolUse` | 1 | `knowledge-capture.js` | Capture file-touch observations to knowledge directory |
| `Stop` | 2 | `session-save.js`, `session-end.js` | Auto-save active session JSON; persist session summary to knowledge |
| `Notification` | 1 | `fleet-notify.js` | Append fleet agent completion events to `notifications.jsonl` |
| `PreCompact` | 1 | `pre-compact.js` | Block compaction if session is mid-critical-task |
| `SubagentStop` | 1 | `subagent-stop.js` | Track subagent cost aggregation per session |

## State Layout

All hook state lives in `~/.claude/commander/` (CCC's own namespace — never touches other `.claude/` contents):

```
~/.claude/commander/
├── sessions/
│   ├── active-session.json       # current session metadata (created on SessionStart)
│   ├── active-cost.json          # incremented on each PreToolUse
│   └── YYYY-MM-DD-{hash}.json    # archived on Stop
├── knowledge/
│   ├── auto-captures.jsonl       # tail of Write/Edit events
│   └── {timestamp}.json          # manual lessons via /ccc-knowledge
├── fleet/
│   └── notifications.jsonl       # fleet agent completion log
├── pre-compact/
│   └── YYYY-MM-DD-{hash}.json    # state saved before compaction
└── config.json                   # user settings (theme, name, etc.)
```

<Note>
  **Privacy:** Hooks never send data off your machine. All state is local. No analytics, no telemetry, no network calls from hooks.
</Note>

## Disabling Individual Hooks

To disable a single hook without uninstalling the plugin, add to `~/.claude/commander/config.json`:

```json
{
  "hooks": {
    "UserPromptSubmit": false,
    "Notification": false
  }
}
```

The hook wrapper checks this flag before running each handler. Disabling a hook event disables all its handlers.

## Hook Execution Model

Hooks are JavaScript files executed natively by Claude Code's hook runtime. They:

1. Receive the event context as JSON on stdin
2. Run their logic (typically 10-50ms)
3. Return `{"continue": true}` or `{"continue": false, "reason": "..."}` on stdout

A `{"continue": false}` return from `PreToolUse` blocks the tool call. All other hooks are informational — they can't block the session flow. This is by design: hooks should enhance, never gate.

See [Architecture](/architecture) for how hooks fit into the full data flow.
