CC Commander ships **17 specialist sub-agents** — all free, no tier gating. Each persona is a scoped execution context: fixed model, specific tools, turn budget, and a voice layer that shapes how it communicates.

This is the **brain/hands** pattern: Claude (your main session) is the brain. Sub-agents are the hands — each specialized for one domain, disposable after the task. Context stays clean because each sub-agent starts fresh.

See [Architecture](/architecture) for how delegation works end-to-end.

## The 17 Personas

<CardGroup cols={2}>
  <Card title="architect" icon="building" color="#8b5cf6">
    **Model:** Opus · **Role:** System design, tradeoff analysis, irreversibility assessment

  <Card title="reviewer" icon="eye" color="#3b82f6">
    **Model:** Sonnet · **Role:** Code review across 4 dimensions: security, performance, correctness, maintainability

  </Card>
  <Card title="builder" icon="hammer" color="#22c55e">
    **Model:** Sonnet · **Role:** Feature implementation, TDD, scaffolding

    Ships the simplest thing that works. Extracts abstractions when seen twice, not once. Highest turn budget for multi-phase builds.
  </Card>
  <Card title="security-auditor" icon="shield" color="#ef4444">
    **Model:** Opus · **Role:** OWASP audits, threat modeling, adversarial review

    Assumes every input is hostile. CWE IDs, exploit concepts, and verified fixes — never hand-waves "probably fine."
  </Card>
  <Card title="debugger" icon="bug" color="#f97316">
    **Model:** Opus · **Role:** Root-cause investigation

    Iron Law: **no fix without confirmed root cause.** Four-phase process: reproduce → hypothesize → verify → fix.
  </Card>
  <Card title="designer" icon="palette" color="#ec4899">
    **Model:** Sonnet · **Role:** UI/UX critique, accessibility, visual hierarchy

    Visual-first. References design principles by name. Never gives vague feedback — specific values, contrast ratios, spacing tokens.
  </Card>
  <Card title="qa-engineer" icon="flask" color="#14b8a6">
    **Model:** Sonnet · **Role:** Edge-case hunting, test coverage audit

    Leads with the breaking case. Enumerates: empty, null, max, overflow, unicode, concurrent. Coverage delta per change.
  </Card>
  <Card title="devops-engineer" icon="server" color="#64748b">
    **Model:** Sonnet · **Role:** Deploy planning, rollback specs, monitoring setup

    Every deployment idea paired with a rollback plan. Talks in SLIs/SLOs/error budgets. Rollback in &lt;5 min or it's incomplete.
  </Card>
  <Card title="data-analyst" icon="chart-bar" color="#0ea5e9">
    **Model:** Sonnet · **Role:** Signal extraction, visualization, statistical honesty

    Shows the data, then the chart, then the insight — in that order. Correlation ≠ causation, always stated.
  </Card>
  <Card title="content-strategist" icon="pen" color="#a855f7">
    **Model:** Sonnet · **Role:** Brand voice, messaging, copy variants

    First question: "Who's the reader?" Offers 3 headline variants. Says "delete this paragraph" when warranted.
  </Card>
  <Card title="product-manager" icon="target" color="#f59e0b">
    **Model:** Opus · **Role:** User stories, acceptance criteria, priority sequencing

    Every feature is a bet. Every bet has a measurable outcome. Scope discipline is a moral act.
  </Card>
  <Card title="performance-engineer" icon="bolt" color="#22d3ee">
    **Model:** Sonnet · **Role:** Hotpath identification, p99 benchmarking

    Measure, don't guess. Every claim backed by a benchmark or labeled "estimated." Never says "faster" without a baseline.
  </Card>
  <Card title="researcher" icon="magnifying-glass" color="#6366f1">
    **Model:** Sonnet · **Role:** Deep research, competitive analysis, citation management

    Facts, inferences, and opinions separated. Every claim cites a source. Recency matters.
  </Card>
  <Card title="technical-writer" icon="file-text" color="#84cc16">
    **Model:** Sonnet · **Role:** Documentation, API reference, clarity audits

    Clarity over cleverness. Active voice. Short sentences. If a beginner can't follow it, it's broken.
  </Card>
  <Card title="fleet-worker" icon="users" color="#06b6d4">
    **Model:** Sonnet · **Role:** Parallel batch work, scoped execution in fleet mode

    Strict-report executor. Stays in scope. Returns structured output the parent can parse. No scope creep.
  </Card>
  <Card title="typescript-reviewer" icon="code" color="#3b82f6">
    **Model:** Sonnet · **Role:** TypeScript-specific code review

    Strict mode compliance, async correctness, ESM/CJS module hygiene, ReDoS detection, prototype pollution, idiomatic type patterns. Severity-rated.
  </Card>

    PEP 8, type hints (PEP 484/604), async/await patterns, pytest fixtures/parametrize, security (SQL/shell injection, path traversal, pickle), idiomatic Python. Severity-rated.
  </Card>

## How Delegation Works

Skills delegate to sub-agents automatically when the task matches their domain:

| Skill | Delegates to | When |
|-------|-------------|------|
| `/ccc-plan` | `product-manager` | Spec interview flow |
| `/ccc-build` | `builder` | After spec approval |
| `/ccc-review` | `reviewer` + `security-auditor` + `qa-engineer` | Diff > 500 lines or multi-dimension review |
| `/ccc-review` (TypeScript) | `typescript-reviewer` | TypeScript files detected in diff |
| `/ccc-review` (Python) | `python-reviewer` | Python files detected in diff |
| `/ccc-design` | `designer` | UI/UX critique or mockup review |
| `/ccc-ship` | `devops-engineer` + `security-auditor` | Pre-flight gate |
| `/ccc-fleet` | `fleet-worker` | Each parallel task spawn |

## Customizing Sub-Agents

Tune behavior via `~/.claude/commander/config.json`:

```json
{
  "agents": {
    "debugger": {
      "maxTurns": 60,
      "fallbackModel": "sonnet"
    },
    "builder": {
      "maxTurns": 80
    }
  }
}
```

All 17 sub-agents are free — no license key, no tier check, no network call required.
