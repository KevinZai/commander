## Positioning

**Aider is your pair programmer. CC Commander is your PM. They're complementary — use aider for solo rapid edits, CCC for multi-phase features that need structure.**

Aider excels at fast, terminal-native edits with multi-LLM flexibility and auto-committed diffs. CC Commander excels at orchestrating the *surrounding process*: planning, specialist review, deploy gates, and memory that compounds across sessions. They solve different problems. Many developers use both.

## 10-Dimension Comparison

| Dimension | aider.chat | CC Commander |
|-----------|-----------|--------------|
| **Surface** | CLI only | Claude Code Desktop plugin + CLI + any MCP-capable IDE |
| **LLM** | Multi (Claude, GPT, DeepSeek, local BYOK) | Claude (user's own subscription) |

| **Agent model** | Single generalist LLM | Brain/hands + specialist sub-agent routing |

| **Cost** | User pays per-token (BYOK) | Free forever (user's existing Claude plan) |
| **Install** | `pip install aider-chat` | `/plugin install commander` in Claude Desktop |
| **Best at** | Fast solo terminal edits, multi-LLM flexibility, auto-committed diffs | Multi-step workflows, review gates, specialist perspectives, structured specs |

## Using Both Together

A common pattern:

1. **`/ccc-plan`** — CC Commander runs the spec interview, writes `tasks/spec-YYYYMMDD.md`
2. **aider** — implements the spec with fast diffs and auto-commits
3. **`/ccc-review`** — CC Commander's reviewer agent audits the diff (security + perf + correctness)
4. **`/ccc-ship`** — CC Commander runs the 22-point pre-flight gate before deploy

Neither tool is "better." They operate at different levels of abstraction.
